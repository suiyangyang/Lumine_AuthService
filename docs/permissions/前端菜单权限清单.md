# Lumine AuthServer 前端菜单权限清单

## 使用建议

- 登录后使用 `/api/auth/login` 返回的权限集合控制前端菜单、按钮和页面访问。
- 推荐将“菜单可见性”和“按钮可操作性”分开判断；菜单通常依赖 `*.view`，写操作按钮依赖 `*.manage`。
- 若接口返回 `403`，可直接读取响应里的 `requiredPermission` 做前端提示。

## 菜单与权限映射

| 菜单/页面 | 建议路由 | 可见权限 | 主要操作权限 | 关联接口 |
| --- | --- | --- | --- | --- |
| 用户管理 | `/system/users` | `users.view` | `users.manage` | `GET/POST/PUT/DELETE /api/users` |
| 用户角色分配 | `/system/users/:id/roles` | `users.view` | `users.manage` | `PUT/POST/DELETE /api/users/{id}/roles...` |
| 角色管理 | `/system/roles` | `roles.view` | `roles.manage` | `GET/POST/PUT/DELETE /api/roles` |
| 角色权限分配 | `/system/roles/:id/permissions` | `roles.view` | `roles.manage` | `PUT/POST/DELETE /api/roles/{id}/permissions...` |
| 权限管理 | `/system/permissions` | `permissions.view` | `permissions.manage` | `GET/POST/PUT/DELETE /api/permissions` |
| 用户组管理 | `/system/user-groups` | `users.view` | `users.manage` | `GET /api/usergroups` |
| OAuth 客户端 | `/auth/clients` | `clients.view` | `clients.manage` | `GET/POST/PUT/DELETE /api/clients` |
| Token 管理 | `/auth/tokens` | `clients.view` | `clients.manage` | `GET /api/tokens`、`POST /api/tokens/{id}/revoke` |
| 菜单管理 | `/system/menus` | `permissions.view` | `permissions.manage` | `GET /api/menus` |
| 审计日志 | `/system/audit` | `clients.view` | `clients.manage` | `GET /api/auditlogs` |

## 前端判断示例

```ts
const canViewUsers = permissions.includes('users.view');
const canManageUsers = permissions.includes('users.manage');

const userMenus = [
  canViewUsers && { name: '用户管理', path: '/system/users' },
].filter(Boolean);

const userActions = {
  create: canManageUsers,
  update: canManageUsers,
  remove: canManageUsers,
  assignRole: canManageUsers,
};
```
