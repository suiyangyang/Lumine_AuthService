# 权限系统当前状态

> 状态：Current
> 更新时间：2026-06-09
> 适用范围：`Lumine.AuthServer` 与 `Lumine.AuthPortal` 当前已落地的权限模型、接口约束和前端联动

本文档描述当前权限系统的可用状态，不再记录单次改造过程。

## 摘要

- 当前系统采用 `用户 -> 角色 -> 权限` 的三级模型。
- 受保护后台接口通过 `[Permission("module.action")]` 声明访问要求。
- 种子数据已内置用户、角色、权限和默认 OIDC 客户端。
- 前端导航已按 `users.*`、`roles.*`、`permissions.*`、`clients.*` 做可见性控制，并扩展到用户组、Token、菜单、审计日志等后台菜单。
- 仪表盘统计也会根据当前账号权限裁剪可见数据。

## 当前权限点

| 模块 | 查看权限 | 管理权限 |
| --- | --- | --- |
| 用户 | `users.view` | `users.manage` |
| 角色 | `roles.view` | `roles.manage` |
| 权限 | `permissions.view` | `permissions.manage` |
| OIDC 客户端 | `clients.view` | `clients.manage` |
| 用户组 | `users.view` | `users.manage` |
| Token 管理 | `clients.view` | `clients.manage` |
| 菜单管理 | `permissions.view` | `permissions.manage` |
| 审计日志 | `clients.view` | `clients.manage` |

这些权限定义来自：

- `Lumine.AuthServer/Infrastructure/Services/AuthSeedOptions.cs`
- 各控制器上的 `[Permission(...)]` 特性

## 当前覆盖范围

### 后端

- `UsersController`：用户列表、详情、个人信息、状态、密码、角色分配
- `RolesController`：角色列表、详情、增删改、权限分配
- `PermissionsController`：权限列表、详情、增删改
- `ClientsController`：OIDC 客户端列表、详情、增删改
- `UserGroupsController`：基于角色聚合的用户组查询
- `TokensController`：refresh token 列表与撤销
- `MenusController`：后台菜单元数据与权限映射
- `AuditLogsController`：授权码、Token 相关审计视图

### 前端

- `MainViewModel` 中的导航菜单会根据权限控制显示
- 管理页面的查询和写操作与对应权限编码保持一致
- 无对应权限时，后端统一返回 `403`，并带 `requiredPermission`

## 当前实现结论

| 项目 | 状态 | 说明 |
| --- | --- | --- |
| 权限编码规范 | 已落地 | 使用小写点分风格，如 `users.view` |
| 接口权限声明 | 已落地 | 后台核心资源接口已标注权限 |
| 种子数据同步 | 已落地 | 默认管理员角色与权限在启动时初始化 |
| 前端菜单联动 | 已落地 | 用户、角色、权限、用户组、客户端、Token、菜单、审计日志页面已接入 |
| 细粒度按钮守卫 | 部分落地 | 主要管理页已接入，后续新增页面需继续同步 |
| 用户组/菜单/审计日志权限 | 已复用现有权限集 | 当前复用 `users.*`、`permissions.*`、`clients.*`，未单独新增权限编码 |

## 维护要求

新增或修改权限时，同步检查：

1. `AuthSeedOptions`
2. 控制器上的 `[Permission(...)]`
3. `docs/api/管理后台基础接口说明.md`
4. `docs/permissions/前端菜单权限清单.md`
5. `Lumine.AuthPortal` 中的导航与页面守卫
