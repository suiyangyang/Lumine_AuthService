# Agent 文档地图

本目录只放面向 coding agent / vibe coding 的维护说明，回答“怎么改这个项目”。

## 文档列表

- [架构维护说明](architecture.md)：项目结构、后端分层、前端联动、认证授权和权限体系边界。
- [代码与文档约定](conventions.md)：命名、分层、接口、权限、数据库、文档维护规则。
- [开发工作流](workflows.md)：常用构建、运行、数据库、迁移、部署和检查命令。

## 使用原则

- 先读根目录 [AGENTS.md](../../AGENTS.md)，再读本目录。
- 改代码时同步维护用户文档；改维护规则时同步维护 agent 文档。
- 文档要服务后续维护，不要把临时聊天记录、一次性思路或未验证假设写成长久规则。
